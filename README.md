## 🛒 커머스 플랫폼 프로젝트

### 📌 소개
Java를 사용하여 만든 콘솔 기반 커머스 플랫폼입니다.
카테고리별 상품 조회, 장바구니 기능, 주문 및 할인 적용, 관리자 상품 관리 기능을 포함하고 있습니다.

### ⚙️ 기능

>사용자 기능

- 카테고리별 상품 조회
- 상품 상세 조회
- 가격대별 상품 필터링 (Stream 사용)
- 장바구니 추가 및 수량 증가
- 장바구니 목록 조회
- 장바구니 상품 삭제
- 주문 확정 및 재고 차감
- 고객 등급별 할인 적용 (Enum 사용)
>관리자 기능

- 관리자 비밀번호 인증
- 상품 추가
- 상품 수정
- 상품 삭제
- 전체 상품 현황 조회

### 🛠️ 기술 스택
- Java
- Stream / Lambda
- Enum
- List (ArrayList)
- 예외처리 (try-catch)

### 📂 구조

프로그램 실행 시작점
전체 객체를 생성하고,
CommerceSystem 실행


## commerce/

#### CommerceSystem.java
- 전체 프로그램 흐름 관리

#### Product.java

- 총 주문 금액 계산 
- 주문 시 재고 차감
#### Admin.java
- 관리자 모드 기능 관리
- 상품 추가 / 수정 / 삭제

#### Grade.java

💡 느낀점

- 객체지향 설계에서 클래스 역할 분리가 중요하다는 것을 배웠습니다 역활분리를 제대로안했다가, 코드가 길어짐에 내가했던 부분이 어딘지 제대로 찾지못해 어려운 부분이있었습니다
- 코드를 전부다 객체지향적으로 설계했다고 장담할순없지만 그래도 객체지향적으로 최대한 하려고 생각하면서 각 클래스의 역활과 정의가 정말중요하고, 시작전에 설계가 정말 중요하다는걸 명확히 파악했습니다
- 처음설계할때 Category,Product,Cart 를 상속이나 인터페이스를 사용하는줄알고 많이 헤멘부분이있었는데 상속하는 것이 아니라 참조하는 구조가 더 자연스럽다는 것을 이해했습니다
- Stream과 Lambda를 활용하여 컬렉션 데이터를 효율적으로 처리하는 방법을 익혔습니다
- Enum을 활용하여 할인 정책을 깔끔하게 관리할 수 있었습니다
- 예외처리를 통해 사용자 입력 안정성을 높이고 프로그램 흐름을 제어하는 방법을 경험했습니다
- Category List, Product List, Cart List 를 통해 각 리스트들간에 관계를 제대로 파악하고,
상위,하위 리스트의 구조를 깨달았습니다
